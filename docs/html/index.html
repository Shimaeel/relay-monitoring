Telnet-SML Documentation

Introduction

Telnet-SML is a substation monitoring system that communicates with relay devices over the Telnet protocol. It uses a Boost.SML finite state machine for connection control and System Event Record (SER) collection.

Key Features

  • State machine orchestration with Boost.SML
  • Configurable authentication retry logic
  • SQLite persistence for SER data
  • Real-time WebSocket server for UI clients
  • SER parsing from relay-formatted responses

Building

The project supports both static executable and DLL builds.

Static Executable (Default)

./build.ps1
# Output: telnet_fsm_test.exe

DLL Library

# Build the DLL
./build.ps1 -DLL
# Output: lib/telnet_sml.dll, lib/libtelnet_sml.a
# Build example that uses the DLL
./build.ps1 -Example
# Output: dll_example.exe

Manual Compilation

# Static build
g++ -std=c++17 -DTELNET_SML_STATIC -I third_party/sml/include \
main.cpp client.cpp ser_database.cpp \
-o telnet_sml -lws2_32 -lmswsock -lsqlite3
# DLL build
g++ -std=c++17 -DTELNET_SML_EXPORTS -shared \
client.cpp ser_database.cpp \
-o telnet_sml.dll -Wl,--export-all-symbols

Usage

// Configure connection
ConnectionConfig conn{"192.168.0.2", 23, 2000ms};
LoginConfig creds{"username", "password"};
RetryState retry{3, 0, 30s};
// Initialize components
TelnetClient client;
SERDatabase db("ser_records.db");
db.open();
SERWebSocketServer wsServer(db, 8765);
wsServer.start();
// Run state machine
sml::sm<TelnetFSM> fsm{client, conn, creds, retry, db};
fsm.process_event(start_event{});
while (!fsm.is("Done"_s) && !fsm.is("Error"_s)) {
fsm.process_event(step_event{});
std::this_thread::sleep_for(200ms);
}
