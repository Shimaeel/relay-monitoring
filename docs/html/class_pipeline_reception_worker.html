Detailed Description

Non-blocking Telnet I/O worker for a single relay pipeline.

Handles TCP connection, login, command execution with retries, and pushes responses to relay-owned ring buffer. Identical in logic to the original ReceptionWorker but used as part of a RelayPipeline.

See also
RelayPipeline Owner of this worker
PipelineProcessingWorker Consumer of the ring buffer

bool PipelineReceptionWorker::driveToOperational ( )
inlineprivate

Drive the RelayConnectionFSM to Operational state.

Repeatedly fires step_event. When the FSM enters a wait state (ConnectWait / LoginRetryWait) the worker sleeps in 1-second chunks so that stop() can interrupt it.

Returns
true FSM reached Operational — ready for commands
false FSM reached Error, or stop was requested

Execute a Telnet command via Command FSM with reconnection.

Up to MAX_RETRIES attempts. Each attempt drives the Connection FSM to Operational, then fires the command through the Command FSM. On failure fires disconnect_event to trigger the reconnection cycle.

◆ handleUserCommand()

std::string PipelineReceptionWorker::handleUserCommand ( const std::string & cmd)
inline

Execute a user-initiated command via Command FSM and return response.

Grabs the command mutex so it doesn't race with the async queue loop, drives Connection FSM to Operational, fires the command through the Command FSM, and returns the raw response text.

Called when user clicks a button in the browser UI.

Parameters
cmdTelnet command (e.g. "SER", "TAR 0", "EVE", "SET ...", "CTRL+C", "CTRL+D", "FIL DIR")
Returns
Raw relay response text, or empty string on failure
