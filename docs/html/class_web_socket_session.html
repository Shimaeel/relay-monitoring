Detailed Description

Handles a single WebSocket client connection.

Manages the lifecycle of a WebSocket session:

  • Accepts WebSocket handshake
  • Sends initial data upon connection
  • Listens for client messages (e.g., "refresh")
  • Responds with current database records as ASN.1 BER/TLV

Uses shared_from_this() pattern for safe async callback handling.

using WebSocketSession::StreamingCallback = std::function<void(const std::string&)>

Callback type for streaming commands (e.g. FETCH_ALL_TAR). Receives the raw command, a callback to send each streamed message, and an abort flag. Returns true if the command was handled as a stream, false to fall through to the normal single-response handler.
