Detailed Description

SQLite database handler for storing and retrieving SER records.

Manages persistent storage of System Event Records using SQLite. Features:

  • Automatic database and table creation
  • Duplicate detection (skips existing record_id + timestamp combinations)
  • Transaction support for bulk inserts
  • Multiple query options (all, by status)
  • JSON export for web UI

Lifecycle

Invariant
db_ is either nullptr (closed) or valid SQLite handle
Table exists after successful open()

bool SERDatabase::createTable ( )
private

Create SER records table and indexes.

Create the ser_records table and indexes if they do not exist.

Creates table with schema:

  • id: Auto-increment primary key
  • relay_id: Relay identifier (e.g., "1")
  • relay_name: Relay display name (e.g., "SEL-751")
  • record_id: Event number from relay
  • timestamp: Event date and time
  • status: Event state (Asserted/Deasserted/empty)
  • description: Event element/description
  • created_at: When record was stored

Also creates indexes on relay_id, record_id, status, and timestamp. Includes migration logic for existing databases without relay columns.

Returns
true if table was created or already exists
false if creation failed

Executes a multi-statement SQL that creates:

  • Table ser_records with columns: id, record_id, timestamp, status, description, created_at. A UNIQUE constraint on (record_id, timestamp) prevents duplicates.
  • Indexes on record_id, status, and timestamp for fast lookups.
Returns
true Schema created or already existed.
false SQL execution failed; last_error_ set.
Precondition
db_ != nullptr (called internally by open()).

bool SERDatabase::open ( )

Open database connection and initialize schema.

Open the SQLite database and create the schema if needed.

Opens SQLite database file and creates table if not exists. Also creates indexes for efficient querying.

Returns
true if database opened and table created successfully
false if open or create failed (check getLastError())
Postcondition
On success: isOpen() == true, table exists
On failure: isOpen() == false, last_error_ contains message
See also
createTable() Internal table creation

Performs the following steps:

  1. If already open, returns true immediately (idempotent).
  2. Calls sqlite3_open() to create/open the database file.
  3. Invokes createTable() to ensure the ser_records table and its indexes exist.
Returns
true Database opened (or was already open) and schema ready.
false sqlite3_open() or createTable() failed. Check getLastError() for details.
Postcondition
On success: db_ != nullptr, isOpen() == true.
On failure: db_ == nullptr, last_error_ set.
See also
close() Release the connection when done.
